Custom native CSS grid and Difficulties I found

  • Relearning css grid properties- trial and error approach, time consuming
  • Potentially harder to scale and maintain dependant on code quality
  • Can't seem to change the order of markup to suit more prominent items. E.g. Impact tracker div is fourth from last on mobile and third from last on tablet, can this be placed second from top so the order would be: event location then impact tracker div without changing markup order? But for desktop, the impact tracker item should be bottom right of the grid so the markup shouldn't change for this scenario.
