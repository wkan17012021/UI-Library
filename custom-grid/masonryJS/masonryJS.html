Masonry JS Library and Difficulties I found

Link Here
  • On page load, there is about 1 to 2 second delay where you can see the layout as a single column left aligned until the JS script kicks in and converts the layout to the mosaic / masonry style. The issue is a bad UX and significant Cumulative Layout Shift. Possibly this can be resolved with strategic management of the script in conjunction with page load, html and css parsing.
  • Images take about 1 to 2 seconds to render on the page. I used the images from UnSplash with large dimensions because they serve as backgrounds to the cells which need to reveal and hide different areas of the image dependnign on what screen size the layout is observed on. This could be resolved by compressing, scaling down images and using lower resolution media. Also, a CDN would be needed to improve page performance.
  • As you drag the window browser to different widths, the page performance is noticeably laggy and may be unattractive to users.
  • On tablet size circa >540px width, there is some whitepace in the bottom right which I couldn't work out how to fill in. It requires trial and error playing around with widths and heights to see what works.
  • The masonry style is based off in-line containers- now that we have CSS Flex and Grid principles, could this be done without a library?
  • Could't work out how to add vertical and horizontal spacing between container cells. The docs do mention a gutter and margin-bottom property but the gutter appeared to move the masonry parent grid element off center and the margin-bottom didn't seem to space vertically the elements- perhaps because they are absolute positioned implemented by the js library?
